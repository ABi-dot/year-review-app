# 外部平台对接策略

## 当前状态

**豆瓣批量导入已完成**，支持：
- RSS 快速导入（最近 10-20 条）
- 全量抓取（书/影/游完整历史，含封面、评分、类型判断）

游戏通过豆瓣游戏库导入，封面和元数据齐全，**无需额外对接 Steam/IGDB**。

## 优先级

### 第一优先级（已完成）

#### 豆瓣（书/影/游）
- **状态**：✅ 已完成批量导入
- **能力**：一键导入读过/看过/玩过的全部历史记录
- **技术方案**：RSS + 网页抓取（grid 模式）+ 移动端 API 补充类型和封面

### 第二优先级（规划中）

#### 搜索联想录入
- **场景**：手动添加单条时，输入书名/影名自动联想
- **能力**：自动填充封面、作者/导演、评分、简介
- **技术方案**：
  1. 优先调用豆瓣公开搜索接口（无需 Key）
  2. 受限时后端代理搜索页解析 HTML（加缓存和限流）
  3. 支持手动上传封面作为 fallback

### 放弃/延后

| 平台 | 原因 |
|------|------|
| Steam | 豆瓣已覆盖游戏导入，无需重复对接 |
| IGDB | 同上，游戏数据已满足需求 |
| 高德/腾讯地图 | 地点改为手动录入，MVP 阶段不自动联想 |
| 小黑盒 | 无开放 API，反爬严格 |
| Netflix / 流媒体 | 无开放 API |

## 数据录入的自动化层级

```
Level 1: 批量导入（豆瓣收藏 → 一键全量同步）✅ 已完成
Level 2: 搜索联想（输入书名/影名 → 自动填元数据）
Level 3: 截图 OCR（电影票根、书店小票拍照识别）
Level 4: 浏览器插件（看书/电影页面时一键收藏）
```

MVP 做到 Level 1-2 即可，Level 3-4 作为后续迭代。
