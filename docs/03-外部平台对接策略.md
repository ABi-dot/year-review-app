# 外部平台对接策略

## 优先级：按数据开放程度排序

### 第一优先级（必接，API 开放）

#### Steam（游戏）
- **开放程度**：完全开放，无需申请
- **能力**：输 Steam ID 可拉取**游戏库 + 游玩时长 + 成就 + 封面**
- **API**：`https://api.steampowered.com/IPlayerService/GetOwnedGames/v1/`
- **难度**：低
- **建议**：优先接入，数据极全

#### IGDB (Twitch)
- **开放程度**：免费，需 Twitch 开发者账号申请 Client ID
- **能力**：游戏封面、评分、类型、开发商、发布日期等元数据
- **难度**：低
- **建议**：配合 Steam 使用，Steam 给库，IGDB 补精美封面和元数据

#### 高德地图 / 腾讯地图（地点）
- **开放程度**：需申请 Key，有免费额度
- **能力**：地点搜索、自动封面图、坐标、分类
- **难度**：低
- **建议**：地点录入时搜索联想，自动带出封面和位置信息

### 第二优先级（需要申请或有风控）

#### 豆瓣（书/影/剧）
- **开放程度**：有公开 API (`api.douban.com/v2`)，但**需申请 API Key**，且对请求频率有限制
- **能力**：搜索书名/电影名 → 返回封面、作者/导演、评分、简介
- **风险**：
  - 无 API Key 时搜索接口可能受限
  - 直接爬网页有风控，需后端代理 + 限流
- **难度**：中
- **建议**：
  1. 先申请豆瓣 API Key
  2. 申请不到则后端做搜索代理，加缓存和限流（1秒/请求）
  3. 同时支持手动上传封面作为 fallback

### 第三优先级（放弃或延后）

#### 小黑盒（游戏）
- **开放程度**：**无开放 API，反爬严格**
- **建议**：放弃，Steam + IGDB 完全可替代

#### Netflix / 其他流媒体
- **开放程度**：几乎无开放 API
- **建议**：放弃，国内以豆瓣影视数据为主

## 数据录入的自动化层级

```
Level 1: 搜索联想（豆瓣/Steam/地图搜索 → 自动填元数据）
Level 2: 一键导入（Steam 游戏库批量导入、豆瓣"看过"列表 CSV 导入）
Level 3: 截图 OCR（电影票根、书店小票拍照 → OCR 识别名称 → 自动匹配）
Level 4: 浏览器插件（看书/电影页面时一键收藏到本系统）
```

MVP 做到 Level 1 即可，Level 2-4 作为后续迭代。
