# 技术路线

## 核心栈选型

| 层级 | 选型 | 理由 |
|------|------|------|
| **前端框架** | Next.js 14+ (App Router) | 全栈一体，API Route 直接写后端，Vercel 一键部署，生态最成熟 |
| **样式** | Tailwind CSS + shadcn/ui | 快速搭建 UI，组件质量高，不用从零设计 |
| **数据库** | PostgreSQL (Supabase) | 关系型数据，免费额度够用；Supabase 有 Dashboard 方便调试 |
| **ORM** | Prisma | 类型安全，迁移方便，和 Next.js 配合好 |
| **文件存储** | Cloudflare R2 | 存封面图，有免费额度，无 egress 费用（比 S3 便宜） |
| **长图生成** | 服务端 Puppeteer (Vercel Serverless) | html2canvas 移动端易翻车，服务端截图最稳；也可客户端用 `html-to-image` 兜底 |
| **AI 文案** | OpenRouter / 硅基流动 / 阿里云百炼 | 聚合平台，按需切换模型，成本低，国内访问友好 |
| **PWA** | next-pwa | 手机端像原生 App，随时记录，可离线查看 |

## 部署方案
- **首选**：Vercel（Next.js 原生支持，自动 CI/CD）
- **数据库**：Supabase（免费档足够）或 Vercel Postgres
- **图片存储**：Cloudflare R2（绑定自定义域名）

## 架构可选升级

### MCP 架构（第二阶段）
可以设计轻量 MCP Server 解耦外部平台：
- `mcp-douban`：标准化返回书籍/电影元数据
- `mcp-steam`：拉取游戏库数据
- 主应用通过 MCP Client 调用，换平台只需换 Server

> 对个人 MVP 来说直接用 Next.js API Route 封装也足够，MCP 可作为后续架构升级。

### 本地优先方案（备选）
如果担心数据隐私或平台依赖：
- 桌面端：Tauri + SQLite + Litestream 自动备份到 S3
- 移动端/ Web：保持不变
