# MVP 里程碑

## 核心原则
先让自己**能用上**，再追求**好用**。每个阶段有明确的"哇塞"时刻。

---

## 阶段 1：最简 CRUD（1-2 周）
**目标**：能记录，能看列表，先解决"有没有"的问题。

### 功能
- [ ] 手动添加条目：书名/电影名/游戏名/地点名
- [ ] 支持上传封面图
- [ ] 打标签（如"力荐"、"治愈"、"烧脑"，可自定义）
- [ ] 评分（1-10 分或五星）
- [ ] 写短评
- [ ] 列表页展示（按时间倒序）
- [ ] 分类筛选（只看书 / 只看游戏）

### 技术任务
- [ ] 初始化 Next.js + Prisma + PostgreSQL 项目
- [ ] 设计核心数据表（Item、Tag、CoverImage）
- [ ] 基础 UI 页面（新增表单 + 列表页）
- [ ] 文件上传到 R2 / 本地（MVP 可先本地）

### 验收标准
每天能无阻碍地记录 1-3 个条目。

---

## 阶段 2：半自动录入（1 周）
**目标**：告别手动传图和填元数据，解决"麻烦"的问题。

### 功能
- [ ] 对接豆瓣搜索 API：输入书名/电影名自动联想
- [ ] 选择后自动填充：封面、作者/导演、评分、简介
- [ ] 对接 Steam：输入 Steam ID 一键导入游戏库
- [ ] 对接 IGDB：补充游戏封面和元数据
- [ ] 对接地图搜索：地点自动联想 + 封面图

### 技术任务
- [ ] 封装豆瓣搜索 API（带缓存）
- [ ] 封装 Steam Web API
- [ ] 封装 IGDB API
- [ ] 前端搜索联想组件

### 验收标准
添加一个条目平均耗时 < 30 秒（从打开页面到保存）。

---

## 阶段 3：年度总结页 + 长图导出（2 周）
**目标**：第一个"哇塞"时刻，解决"总结缺失"的问题。

### 功能
- [ ] 年度总览页：统计数字（看过 X 本书、Y 部电影...）
- [ ] 时间线视图：按月/季度展示消费轨迹
- [ ] 标签云 / 类型分布图
- [ ] 长图生成：支持多种模板
  - 朋友圈模板（9:16，重设计）
  - 小红书模板（3:4，带文案区）
  - 微博模板（16:9，头图风格）
- [ ] 长图支持下载为 PNG

### 技术任务
- [ ] 设计总结页 HTML/CSS 模板
- [ ] Puppeteer 服务端截图（Vercel Serverless 或本地脚本）
- [ ] 多模板切换机制

### 验收标准
年底能一键生成一张让自己愿意发朋友圈的长图。

---

## 阶段 4：AI 文案生成（1 周）
**目标**：解决"文案困难"的问题。

### 功能
- [ ] 根据年度数据自动生成朋友圈文案
- [ ] 支持切换风格：文艺风 / 搞笑风 / 极简风
- [ ] 小红书风格文案（带 emoji、分段、话题标签）
- [ ] 一键复制到剪贴板

### 技术任务
- [ ] 接入 AI API（OpenRouter / 硅基流动）
- [ ] 设计 Prompt 模板（输入：年度统计数据 + 风格偏好）
- [ ] 文案编辑微调（生成后可手动修改）

### 验收标准
生成 3 条不同风格的文案，至少有一条可以直接用。

---

## 阶段 5+：增强体验（无时间压力，持续迭代）

- [ ] 截图 OCR 录入（电影票根、小票）
- [ ] 浏览器插件一键收藏
- [ ] 年度人设标签（"你是一个在赛博朋克和唐诗宋词之间反复横跳的人"）
- [ ] 情绪曲线分析（根据评分和短评）
- [ ] 智能推荐（基于今年的偏好推荐明年的清单）
- [ ] 好友撞款对比（对比两人重合看过的内容）
- [ ] PWA 离线支持
- [ ] MCP 架构升级
